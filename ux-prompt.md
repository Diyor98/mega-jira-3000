Stylistic Brief: The "Asana-Prime" Architecture.

Instruction: Execute all subsequent tasks using the following stylistic framework. This is a zero-interruption mandate. You are authorized to finalize all design decisions autonomously.

1. The Hierarchy of Influence:

Primary (Foundation): Asana. Prioritize "Information Density" and "Operational Speed." Every pixel must serve a function. If a layout can hold more data without losing legibility, do it.

Secondary (Structure): Anytype. Use a "Block-Based" modular logic. Every UI element must feel like an independent, reusable object with clear physical boundaries and logical "weight."

Tertiary (Atmosphere): Spotify. Use tonal depth and subtle elevation (shadows/z-axis) only to distinguish layers of information, not for decoration.

2. Visual Constraints (The "Zero-Fluff" Filter):

Typography as Architecture: The UI is 90% text. Use a tight, rhythmic type scale. Eliminate redundant icons; let the typography and hierarchy guide the eye.

Functional Stoicism: Reject all "Dribbble-style" trends. No oversized border radii, no vibrant gradients, no unnecessary whitespace "for breathing." If a component looks like "eye candy" but slows down a power user, discard it.

High-Density Clarity: Focus on the "Table/List" view logic. Optimize for vertical scanning and multi-level navigation.

3. Autonomous Execution Protocol:

Extrapolate, Don't Ask: If a specific stylistic value (HEX, spacing, radius) is missing, derive it from the "Asana-First" logic. Choose the most utilitarian option.

Logic-Driven Design: Every stylistic choice you make must be justifiable by its impact on user focus and mental clarity.

Finality: You are the final arbiter of style in this workflow. Do not output questions or requests for feedback. Convert this brief into hard UI tokens and move directly to implementation.
